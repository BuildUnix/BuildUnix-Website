# Contributing to BuildUnix

Thank you for your interest in contributing to the BuildUnix marketing website.

## Workflow

1.  **Atomic Commits**: We prefer small, logical commits that do one thing well.
2.  **Code Style**: We use Prettier for formatting and CSS Modules for styling.
3.  **3D Components**: When modifying 3D scenes, ensure performance remains a priority (limit draws, optimize materials).

## Commit Message Convention

We follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:
- `feat:` for new features.
- `fix:` for bug fixes.
- `docs:` for documentation changes.
- `refactor:` for code changes that neither fix a bug nor add a feature.
- `chore:` for maintenance tasks.

## Getting Started

1.  Fork the repository.
2.  Install dependencies: `npm install`.
3.  Create a branch for your changes.
4.  Submit a Pull Request.
